emit

a Deferred's promise differs from a normal event Emitter because it has two states: "waiting" and "resolved". When the promise is waiting, all event emissions are queued. When the promise is either resolved or rejected, all of those events are emitted. All subsequent emissions are sent immediately.

either a promise or the resolved value for the promise.
