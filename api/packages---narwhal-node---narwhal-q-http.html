Server

respond respond(request Request)

a JSGI responder function that receives a Request object as its argument. The JSGI responder promises to return an object of the form {status, headers, body}. The status and headers must be fully resolved, but the body may be a promise for an object with a forEach(write(chunk String)) method, albeit an array of strings. The forEach method may promise to resolve when all chunks have been written.
returns undefined

a Node Server object.

Client

Creates an HTTP client for issuing requests to the given host on the given port.

port Number
host String

Properties:

request

Issues an HTTP request. The request may be any object that has method, path, headers and body properties.

  • method String is optional, defaults to "GET".
  • path String is optional, defaults to "/".
  • headers Object is optional, defaults to {}.
  • body is optional, defaults to []. Body must be an object with a forEach method that accepts a write callback. forEach may return a promise, and may send promises to write. body may be a promise.

The Q HTTP Server responder receives a Request object that is suitable for request, and request returns a Response suitable for returning to the Server.
