fail

Called polymorphically by assert objects, particularly this assert module, when a test fails. The behavior of this implementation of fail is to throw an AssertionError, but by prototypically inheriting the assert module, using Object.create or its ilk, and overriding fail, it is possible to trivially create variations that log failures rather than throwing. The test module sends such an object to each test* method in a unit test tree so that tests have the option of logging instead of throwing. Because each test receives a different assert object, the logs are associable with the individual test.
